# 管理后台实施计划

## 当前状态

- 技术栈：Next.js App Router、TypeScript、Vercel、Neon PostgreSQL。
- 已完成：生产数据库资源、20 项可重复执行迁移、产品/新闻/询盘 PostgreSQL 读写层、Google Search Console 服务端读取接口。
- 旧的 `data/local-site-data.json` 仅作为本地迁移兼容来源，生产环境不再依赖它保存业务数据。

## 实施阶段

1. 基础架构：数据库迁移、账户与会话、角色权限、审计日志、统一 API 错误格式。
2. 内容管理：产品分类、产品状态与 SEO 字段、新闻分类、媒体库与发布流程。
3. 客户线索：服务端分页、筛选、状态流转、负责人、备注、导出、反垃圾与通知。
4. 数据与 SEO：仪表盘、访问事件、SEO 问题中心、同步中心、同步日志与定时任务。
5. 加固交付：测试、速率限制、备份/恢复、部署/运维/API 文档。

## 安全与风险

- 密码、Google 服务账号和数据库连接均只允许通过 Vercel 加密环境变量传递。
- 媒体库需要 Vercel Blob 或等价对象存储；邮件通知、验证码和第三方统计需要各自的凭证后才能启用。
- 每一项生产写入功能先实现数据库约束与审计，再开放后台操作入口。
