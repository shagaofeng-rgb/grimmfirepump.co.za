# 管理后台与新闻自动化实施计划

## 当前状态

- 技术栈：Next.js App Router、TypeScript、Vercel、Neon PostgreSQL。
- 已完成：生产数据库资源、20 项可重复执行迁移、产品/新闻/询盘 PostgreSQL 读写层、Google Search Console 服务端读取接口。
- 旧的 `data/local-site-data.json` 仅作为本地迁移兼容来源，生产环境不再依赖它保存业务数据。

## 新闻自动化（2026-07-13）

- 审计结果：原 `/news` 为手工示例；没有来源采集、任务表、定时任务、72 小时校验、7 天去重、产品关系、RSS 或 News Sitemap。因此不能称为“自动发布”。
- 数据层：为新闻补充来源可追溯、SEO/GEO、状态字段；新增 `news_sources`、`news_candidates`、`news_product_relations`、`news_jobs` 与 `news_publication_audits`，并用唯一幂等键防止并发重复运行。
- 来源：仅采集管理员通过 `NEWS_SOURCE_FEEDS` 配置的公开 HTTPS RSS/Atom 源；拒绝私网、未白名单域、超大响应和无明确发布时间的条目。来源内容仅作为事实摘要输入，绝不保存或转载全文。
- 质量门槛：原始发布时间 72 小时内、最近 7 天 URL/指纹去重、至少一个已发布真实产品关联、相关性阈值、来源与发布时间完整。未通过的内容会保留为候选/审核记录，绝不发布。
- 生成与发布：没有可核验的内容生成服务时仅生成 `review_required` 草稿；不得为完成每日目标而自动发表。配置 `NEWS_AUTO_PUBLISH=true` 后仍需通过所有质量校验。
- 调度：Vercel Cron 每 3 小时调用受 `NEWS_CRON_SECRET` 保护的路由；任务按日/小时幂等、记录审计和失败原因，缺口会在下一次窗口补发。
- 关联与 SEO：新闻—产品关系单独存储，新闻页与产品页双向展示；发布时重新验证 news/products 标签；新增 News Sitemap 与 RSS Feed。
- 外部凭证：必须由运营方配置可信 RSS 源；如要自动生成而非审核草稿，还必须配置受支持且已授权的 AI 服务。当前不会使用 Google Search Console 凭据生成新闻。

## 实施阶段

1. 基础架构：数据库迁移、账户与会话、角色权限、审计日志、统一 API 错误格式。
2. 内容管理：产品分类、产品状态与 SEO 字段、新闻分类、媒体库与发布流程。
3. 客户线索：服务端分页、筛选、状态流转、负责人、备注、导出、反垃圾与通知。
4. 数据与 SEO：仪表盘、访问事件、SEO 问题中心、同步中心、同步日志与定时任务。
5. 加固交付：测试、速率限制、备份/恢复、部署/运维/API 文档。

## 新闻自动化进度

1. 已完成：审计、迁移设计、来源与安全规范、前端/SEO/调度实施计划。
2. 进行中：数据库迁移、RSS 候选采集、质量门槛、新闻—产品关系、Cron 与后台接口。
3. 待验收：配置生产 RSS 来源和（如需自动生成）内容服务后，观察首个自然日审计；没有真实合格来源时系统应告警而非凑数发布。

## 安全与风险

- 密码、Google 服务账号和数据库连接均只允许通过 Vercel 加密环境变量传递。
- 媒体库需要 Vercel Blob 或等价对象存储；邮件通知、验证码和第三方统计需要各自的凭证后才能启用。
- 每一项生产写入功能先实现数据库约束与审计，再开放后台操作入口。
